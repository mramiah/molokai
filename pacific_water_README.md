# Pacific Water SCADA Graph Database

A comprehensive graph database implementation for the water distribution system of Pacific Water, serving the city of West Pacific (population 10,000).

## 📊 Project Overview

This project models a complete SCADA (Supervisory Control and Data Acquisition) system as an interconnected graph of components, visualizing how water flows through treatment, storage, and distribution infrastructure while being monitored and controlled by SCADA systems.

### System Characteristics
- **Service City**: West Pacific
- **Population**: 10,000
- **Treatment Capacity**: 5,000 gallons per minute (gpm)
- **Distribution Zones**: 4 (Residential North, Commercial Central, Industrial East, Residential South)
- **Communication Backbone**: PacCom fiber/5G network supporting SCADA traffic
  - hubs: Central Gateway (`paccom_001`), West Hub (`paccom_002`), East Tower (`paccom_003`)
- **Network Components**: 32 nodes (+3 PacCom hubs)
- **Network Connections**: 35 edges (+6 communication links between servers/RTUs and hubs, +2 inter-hub links)

## 🏗️ System Architecture

```

        ↓
    Distribution Mains (3)
    ├→ North Main (12" diameter)
    ├→ Central Main (14" diameter)

        ↓

    ├→ North Residential (2,500 customers, 800 gpm)
    ├→ Central Commercial (1,500 customers, 600 gpm)
    ├→ East Industrial (800 customers, 500 gpm)
    └→ South Residential (2,200 customers, 700 gpm)
```

## 🔗 Connection Types

### Hydraulic Connections (Water Flow)
- **Count**: 15
- **Direction**: Unidirectional
- **Properties**: Flow rate, diameter, length, material
- **Example**: Treatment Plant → Pumping Station

### Control Connections (SCADA Commands)
- **Count**: 10
- **Direction**: Bidirectional
- **Properties**: Priority level, protocol
- **Example**: SCADA Server → RTU Controller

### Monitoring Connections (Data Collection)
- **Count**: 5
- **Direction**: Unidirectional (sensor → controller)
- **Properties**: Data type, frequency
- **Example**: Flow Meter → RTU

```python
paths = scada.find_paths("tp_001", "cz_001")
```

### 2. Get All Components Under SCADA Control

```python
neighbors = scada.get_neighbors("scada_srv_001")
```

```python
total = sum(c.capacity for c in scada.components.values() if c.capacity)

```

### 4. Find Bottlenecks
```python
degrees = dict(scada.graph.degree())
critical = sorted(degrees.items(), key=lambda x: x[1], reverse=True)[:5]

```

```python
for node1, node2 in combinations(scada.graph.nodes(), 2):

```

## 🗄️ Neo4j Integration

### Setup Neo4j Database

1. **Install Neo4j Community Edition**
   - Download from: https://neo4j.com/download/
   - Install locally or use Docker

2. **Start Neo4j Service**
   ```bash
   # macOS with Homebrew
   brew services start neo4j
   
   # Docker
   docker run --name neo4j -p 7474:7474 -p 7687:7687 neo4j:latest
   ```

3. **Load Graph Data**
   - Open Neo4j Browser (http://localhost:7474)
   - Execute commands from `pacific_water_scada_neo4j.cypher`

4. **Run Queries**
   ```cypher
   // Find all components connected to treatment plant
   MATCH (tp:treatment_plant {id: 'tp_001'})-[*]-(component)
   RETURN DISTINCT component.name

   // Get system statistics
   MATCH (n) RETURN labels(n)[0] as type, count(*) as count

   // Trace water flow path
   MATCH p = (tp:treatment_plant)-[:FLOW_TO*]->(cz:customer_zone)
   RETURN p
   ```

## 🛡️ Resilience & Redundancy

### Current Strengths
✅ **Dual Pumping Stations** - Provides partial redundancy
✅ **Two Storage Reservoirs** - Distributed storage reduces single-point failure risk
✅ **Multiple Distribution Mains** - Allows route diversity
✅ **Redundant Monitoring** - Multiple RTUs provide control backup

### Known Single Points of Failure
⚠️ **Treatment Plant** - Only one source, no backup facility
⚠️ **Central SCADA Server** - No backup control center
⚠️ **Central Distribution Main** - Handles 30% of load without alternative

### Recommended Improvements
1. **Add backup treatment source or emergency water connection**
2. **Implement redundant SCADA systems with automatic failover**
3. **Install cross-connection valve between distribution mains**
4. **Add tertiary pumping station with auto-switchover**
5. **Implement distributed SCADA servers (edge computing)**
6. **Add backup power supply (UPS/generator) for critical components**

## 🔄 Graph Operations

### Core Methods in SCADAGraphDatabase

```python
# Add component to graph
scada.add_component(component)

# Add connection between components
scada.add_connection(connection)

# Retrieve component details
scada.get_component(component_id)

# Find connected components
scada.get_neighbors(component_id)

# Find all paths between two components
scada.find_paths(source_id, target_id)

# Get subgraph by component type
scada.get_subgraph_by_type(component_type)

# Get system statistics
scada.get_graph_statistics()

# Export to JSON
scada.to_json()

# Export to Neo4j Cypher
scada.export_to_neo4j_cypher()
```

## 🛠️ Extending the System

### Add New Customer Zone
```python
new_zone = Component(
    id="cz_005",
    name="New Development Zone",
    component_type=ComponentType.CUSTOMER_ZONE,
    location="New area",
    latitude=21.31,
    longitude=-157.85,
    metadata={"customers": 500, "avg_consumption_gpm": 200}
)
scada.add_component(new_zone)

# Connect to distribution main
connection = Connection(
    source_id="dm_001",
    target_id="cz_005",
    connection_type=ConnectionType.HYDRAULIC,
    flow_direction="unidirectional",
    capacity=200
)
scada.add_connection(connection)
```

### Add New Monitoring Point
```python
new_sensor = Component(
```

## 🐛 Troubleshooting

### ImportError: No module named 'networkx'
```bash
pip install networkx matplotlib
```

### Graph visualization not generating
- Ensure `matplotlib` is installed: `pip install matplotlib`
- Check file permissions for output directory
- Verify enough disk space available

### Neo4j connection failed
- Ensure Neo4j service is running
- Check Neo4j browser at http://localhost:7474
- Verify connection credentials in config

### Memory issues with large graphs
- Reduce graph size or use sampling for visualization
- Use Neo4j for production (handles larger datasets efficiently)
